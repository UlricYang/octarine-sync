# LeetCode 287 - 寻找重复数

## Step 1: 题目描述

给定一个包含 `n + 1` 个整数的数组 `nums`，其数字都在 `[1, n]` 范围内（包括 1 和 n），可知至少存在一个重复的整数。

假设 `nums` 只有一个重复的整数，返回这个重复的数。

你必须设计并实现一个**线性级时间复杂度** `O(n)` 的解决方案，并且**只能使用常量级额外空间** `O(1)`。

**示例 1**：
输入：`nums = [1,3,4,2,2]`
输出：`2`

**示例 2**：
输入：`nums = [3,1,3,4,2]`
输出：`3`

**示例 3**：
输入：`nums = [3,3,3,3,3]`
输出：`3`

**约束条件**：

- `1 <= n <= 10^5`
- `nums.length == n + 1`
- `1 <= nums[i] <= n`
- `nums` 中只有一个整数出现**两次或多次**，其余整数均只出现一次。

## Step 2: 核心结论（金字塔结构）

### 核心结论

本题的最优解是**Floyd判圈算法（龟兔赛跑算法）**，其核心优势在于：**在满足 O(n) 时间复杂度和 O(1) 空间复杂度的前提下，将数组抽象为链表，利用环检测找到重复数字**。

### 支撑论点（MECE 分类）

#### A. 理论最优性：Floyd算法的完备性

- **问题本质**：这本质上是一个**图论中的环检测问题**。将数组下标和值构成的映射 `i -> nums[i]` 看作链表，重复数字意味着链表中存在环。
- **关键洞察**：
  - 将数组看作一个函数 `f(i) = nums[i]`，从任意起点出发，按照 `x = f(x)` 迭代，会进入一个环。
  - 重复的那个数正是环的入口点。
- **Floyd算法核心思想**：
  1. **Phase 1（环检测）**：使用快慢指针找到环内的相遇点。
  1. **Phase 2（环入口定位）**：将一个指针重新放到起点，两个指针同步移动，相遇点即为环入口（重复数）。

#### B. 对比劣势性：其他方法的缺陷

| 方法             | 时间复杂度 | 空间复杂度 | 缺陷分析                   |
| ---------------- | ---------- | ---------- | -------------------------- |
| 哈希表           | O(N)       | O(N)       | 违背“常量空间”要求         |
| 排序法           | O(N log N) | O(1)       | 违背“线性时间”要求         |
| 二分查找（值域） | O(N log N) | O(1)       | 虽然满足空间，但时间不达标 |
| **Floyd算法**    | **O(N)**   | **O(1)**   | **满足全部约束，理论最优** |

#### C. 适用边界：明确约束与扩展性

- ✅ 适用：数组长度为 n+1，元素范围 [1,n]，仅一个重复数。
- ⚠️ 需调整：若有多个重复数 → Floyd仍能找到一个，但无法找出全部。
- ⚠️ 需调整：若允许修改数组 → 可用原地标记法（取负数）。

#### D. 工程实践价值：面试评分标准

- ✅ **算法功底**：掌握图论和Floyd算法。
- ✅ **抽象能力**：能将数组抽象为链表结构。
- ✅ **数学证明**：能解释为何算法正确。
- ✅ **约束敏感**：能准确理解并满足时间和空间复杂度要求。

### 总结

因此，**Floyd判圈算法**是本题在理论和工程约束下的最优解，体现了深刻的数学洞察力。

## Step 3: 多语言实现

### Go 🐹

```go
package main

// findDuplicate 寻找数组中的重复数
// 使用 Floyd 判圈算法（龟兔赛跑）
// 时间复杂度: O(n), 空间复杂度: O(1)
func findDuplicate(nums []int) int {
    // Phase 1: 环检测 (Floyd's Cycle Detection)
    // 初始化快慢指针，都从索引 0 开始
    slow, fast := nums[0], nums[0]

    // 移动指针直到相遇，证明存在环
    // 慢指针每次走一步: slow = nums[slow]
    // 快指针每次走两步: fast = nums[nums[fast]]
    for {
        // 慢指针移动一步
        slow = nums[slow]
        // 快指针移动两步
        fast = nums[nums[fast]]

        // 当快慢指针相遇时，说明已进入环中
        if slow == fast {
            break
        }
    }

    // Phase 2: 寻找环的入口点（重复的数字）
    // 将其中一个指针重新指向起点
    slow = nums[0]

    // 两个指针以相同速度移动，再次相遇的点即为环入口
    for slow != fast {
        slow = nums[slow]
        fast = nums[fast]
    }

    // 返回环入口的值，即重复的数字
    return slow
}
```

#### 算法深入解析（费曼式三层结构）

**第一层：一句话讲明白**

> 把数组想象成一个跑道，每个数字告诉你下一步去哪个位置。因为有重复数字，所以跑道会出现一个圈。你派出两个人，一个跑得快一个跑得慢。他们迟早会在圈里碰头。然后，让其中一个人回到起点，两人再以一样速度跑，下次碰头的地方，就是那个重复数字的位置。

**第二层：手把手教你写**

- **为什么可以把数组看成链表？**
  - 因为 `nums[i]` 的值在 `[1,n]` 范围内，正好可以作为下一个索引。
  - 例如 `nums[0] = 3`，就可以跳到 `nums[3]`，就像链表的 `next` 指针。

- **为什么一定会形成环？**
  - 因为数组长度是 `n+1`，但值域是 `[1,n]`。根据鸽笼原理，必有重复值。
  - 设重复值为 `x`，那么 `nums[i] = x` 和 `nums[j] = x`，就会有两个指针指向同一个位置，从而形成环。

- **Phase 1 为什么能找到环？**
  - 快指针速度是慢指针的2倍，在环里跑，快指针迟早会追上慢指针。
  - 这是 Floyd 算法的经典应用。

- **Phase 2 为什么能找到入口？**
  - 这是 Floyd 算法的精华。设：
    - 起点到环入口距离为 `a`
    - 环入口到相遇点距离为 `b`
    - 相遇点再到环入口距离为 `c`
  - 相遇时，慢指针走了 `a+b`，快指针走了 `a+b+c+b = a+2b+c`。
  - 因为快指针是慢指针的2倍速度，所以 `2(a+b) = a+2b+c`，得出 `a = c`。
  - 因此，一个指针从起点走 `a` 步，另一个从相遇点走 `c` 步，两者必在入口相遇。

**第三层：为什么这样最好**

- **设计哲学**：
  - 这是将离散数学问题（数组）映射到连续数学模型（图论）的典范。
  - 体现了“万物皆图”的深刻思想。

- **工程优势**：
  - **极致的空间效率**：只用了两个指针变量。
  - **严格的复杂度控制**：满足面试官苛刻的要求。
  - **广泛的适用性**：Floyd 算法可用于任何环检测场景。

- **面试加分点**：
  - 能手推 Floyd 算法第二阶段的数学证明。
  - 能指出其与哈希表法在时空权衡上的根本差异。
  - 能讨论其在分布式系统中的应用（如检测死锁）。

### Python 🐍

```python
class Solution:
    def findDuplicate(self, nums: List[int]) -> int:
        """
        使用 Floyd 判圈算法寻找重复数
        :param nums: 包含 n+1 个整数的数组，数字在 [1, n] 范围内
        :return: 重复的整数
        """
        # Phase 1: 环检测
        slow = fast = nums[0]
        while True:
            slow = nums[slow]
            fast = nums[nums[fast]]
            if slow == fast:
                break

        # Phase 2: 寻找环入口
        slow = nums[0]
        while slow != fast:
            slow = nums[slow]
            fast = nums[fast]

        return slow
```

## Step 8: 面试追问

### Q1: 为什么不能用哈希表？

**标准回答**：题目要求空间复杂度 O(1)，哈希表需要 O(N) 空间。
**加分回答**：若无空间限制，哈希表是最直观高效的解法，只需一轮遍历。

### Q2: 如果允许修改数组，还有哪些解法？

**标准回答**：可以将 `nums[abs(x)]` 取负来标记已访问，若再次遇到负数则 `abs(x)` 是重复数。
**加分回答**：这是“原地标记法”，但违背了“不修改数组”的隐含要求。

```python
# 原地标记法（仅供思路参考）
def findDuplicate(nums):
    for num in nums:
        index = abs(num)
        if nums[index] < 0:
            return index
        nums[index] = -nums[index]
    return -1
```

### Q3: Floyd算法的Phase 2数学证明能再讲一遍吗？

**标准回答**：详见Step 3的算法深入解析。
**加分回答**：能用图示和变量清晰表达 `a = c` 的推导过程。

### Q4: 为什么不用二分查找？

**标准回答**：二分查找值域需要统计小于mid的元素个数，时间复杂度为 O(N log N)。
**加分回答**：虽然满足空间要求，但在时间上不达标。

```python
# 二分查找法（时间复杂度 O(N log N)）
def findDuplicate(nums):
    low, high = 1, len(nums) - 1
    while low < high:
        mid = (low + high) // 2
        count = sum(1 for x in nums if x <= mid)
        if count > mid:
            high = mid
        else:
            low = mid + 1
    return low
```

### Q5: 如果有多个重复数，此算法能找到哪一个？

**标准回答**：能找到其中一个，但无法保证是哪一个。
**加分回答**：Floyd算法保证能找到环中的一个点，但不保证入口点。

### Q6: 此算法与快排的partition思想有何异同？

**标准回答**：完全不同。Partition是分治思想，Floyd是图论思想。
**加分回答**：能指出两者在“指针移动”上的表面相似性，但本质逻辑完全不同。

### Q7: 如何证明数组一定有环？

**标准回答**：根据鸽笼原理，n+1个数放入n个位置，必有冲突。
**加分回答**：能从函数映射的角度分析不动点和环的存在性。

### Q8: 有哪些常见的边界测试用例？
